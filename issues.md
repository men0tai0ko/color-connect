## 解決済み

### [FIXED v1.8.1] 手動シャッフル: コンボリセット漏れ
- **現象:** 手動シャッフル使用時にコンボがリセットされずスコアに不正加算
- **原因:** `_doManualShuffle()` に `GameState.resetCombo()` / `UIManager.updateCombo(0)` が未実装
- **修正:** 上記2行を追加（`cancelChain()` と同等の処理）

### [FIXED v1.8.1] 手動シャッフル中のendGame競合
- **現象:** シャッフルアニメーション中にタイムアップすると `_animating` が二重解除
- **原因:** `endGame()` が `_animating = false` を設定した後、`_doManualShuffle()` の `await` 完了後にも `false` を再設定
- **修正:** ペナルティで `timeLeft <= 0` の場合は即 `endGame()` 委譲。`_doShuffle()` 完了後に `!GameState.running` チェックで早期 return

### [FIXED v1.6.3] hasValidMove WC異色判定バグ
- **現象:** 🔴★🔵🔵🔵 のような異色チェーンを「有効手なし」と誤判定しシャッフルが過剰発動
- **原因:** BFS の `nextActive` がWC通過後も起点色のまま保持され、WC後の異色ブロック群を辿れなかった
- **修正:** WC通過後の通常ブロック色を `nextActive` として正しく更新

### [FIXED v1.5.1] ワイルドカード消去判定バグ（4件）
- 先頭タップ確定でnormalCount確認なし
- チェーンバーの有効表示がchain.lengthで判定
- 詰まり検出BFSがWCを考慮しない

### [FIXED v1.5.3] WCブロックがほぼ黄色単色に見える
- **原因①:** background-position アニメーションが黄色帯付近で止まって見えた
- **原因②:** `::before` / `::after` 疑似要素がグラデーションを覆っていた
- **修正:** `hue-rotate` アニメーションに変更、疑似要素を無効化、★マーク追加

### [P2] 手動シャッフル使用可能閾値
- **内容:** 残り11秒でシャッフルすると残り1秒になりほぼゲームオーバー確定になる
- **現状:** 残り10秒以下で無効（ボタングレーアウト）
- **検討案:** 閾値を `PENALTY * 2`（20秒以下で無効）に変更
- **優先度:** P2（UX改善・プレイに支障なし）
- **参照:** v1.8.0 QAレビュー
