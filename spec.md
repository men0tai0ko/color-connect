## 4. ゲームルール

- 制限時間内にブロックをチェーンして消去しスコアを競うタイムアタック形式
- ゲーム開始時に **3→2→1→GO!** のカウントダウン演出が流れる（3.0秒）
  - カウントダウン中はグリッドを表示するが操作は無効
  - GO! 表示と同時にブロック操作が有効化される（フェードアウトは背後で継続）
- グリッド上のブロックをドラッグ・スワイプで繋げて選択し、離すと消去確定
- 通常ブロックは **同色かつ隣接（上下左右）** するものしか繋げられない
- チェーンを確定するには **通常ブロックが3個以上** 必要
- 消去後は上のブロックが落下し、空いたマスに新しいブロックが補充される

## 5. 難易度設定

| 難易度 | 制限時間 | 使用色数 | スコア倍率 |
|---|---|---|---|
| EASY | 90秒 | 4色 | ×0.8 |
| NORMAL | 60秒 | 5色 | ×1.0 |
| HARD | 45秒 | 5色 | ×1.5 |

- デフォルト難易度は NORMAL
- 難易度はタイトル画面でゲーム開始前に選択
- ベストスコアとランキングは難易度ごとに独立して保存される

## 6. スコアリング

### 基本スコア（チェーン数に応じた加算）

| チェーン数 | 基本スコア |
|---|---|
| 3個 | 100点 |
| 4個 | 250点 |
| 5個 | 450点 |
| 6個以上 | 450 ＋ (n－5) × 100点 |

### ボーナス・倍率（乗算順）

1. **アーリーボーナス**: 残り時間が制限時間の50%以上のとき +20点
2. **ラストスパート**: 残り10秒以下のとき ×1.5倍
3. **難易度倍率**: EASY×0.8 / NORMAL×1.0 / HARD×1.5
4. **コンボボーナス**: 2コンボ目から +30%ずつ増加、最大×3.0倍

> 計算順: 基本スコア → アーリーボーナス加算 → ラストスパート倍率 → 難易度倍率 → コンボ倍率

## 7. チェーン選択操作

### 選択方法

| 操作 | 動作 |
|---|---|
| ドラッグ（PC: マウス / スマホ: スワイプ） | なぞったブロックを順次追加 |
| ドラッグ終了（指・ボタンを離す） | 通常ブロック3個以上 → 自動確定、未満 → 自動キャンセル |
| 1つ前のブロックへ戻る | チェーン末尾を取り消し（巻き戻し） |
| チェーン先頭ブロックをタップ | 通常ブロック3個以上 → 確定、未満 → キャンセル |

### 制約

- ワイルドカード単体ではチェーンを開始できない
- 同じブロックを2度選択することはできない（ループ禁止）
- 選択可能な隣接ブロックは「到達可能（reachable）」として点滅ハイライト表示

## 8. ワイルドカードブロック

### 概要

通常ブロックとは異なる特殊ブロック。レインボーグラデーション＋★マークで表示される。

### 出現ルール

- 残り時間が制限時間の **75% / 50% / 25%** に達したタイミングで出現
- 1回の出現につき **2個同時** に変化
- グリッド上の通常ブロックがランダムにワイルドカードへ変化する
- **上限: 6個**（グリッド上に同時に存在できる最大数）
- 上限に達している場合は出現しない

#### 出現位置の優先順位

以下の優先度で出現位置を決定する（高優先度の候補がない場合は次の優先度へ）。

| 優先度 | 条件 |
|---|---|
| 1位 | 既存WCとの距離3以上 かつ 橋渡しで通常ブロック3個チェーン成立 |
| 2位 | 既存WCとの距離2以上 かつ 橋渡しで通常ブロック3個チェーン成立 |
| 3位 | 既存WCとの距離3以上 |
| 4位 | 既存WCとの距離2以上 |

- 選択中のブロックは変化対象から除外
- 既存WCのマスは変化対象から除外

### チェーンへの組み込みルール

| 操作 | 可否 |
|---|---|
| WC単体でチェーン開始 | ✕ 不可 |
| 通常ブロックのチェーンにWCを追加 | ✓ 可（隣接していれば） |
| WCの後に **同色** を追加 | ✓ 可 |
| WCの後に **異色** を追加 | ✓ 可（色の切り替えが可能） |
| WCを複数含むチェーン | ✓ 可（制限なし） |

### 消去条件

- チェーン内の **通常ブロックが3個以上** あれば消去可能
- WCのみ、またはWCと通常ブロック2個以下の組み合わせは消去不可

| 構成例 | 通常ブロック数 | 消去 |
|---|---|---|
| 🔴🔴🔴 | 3 | ✓ 可 |
| 🔴🔴★🔵 | 3 | ✓ 可 |
| 🔴🔴★🔵🔵 | 4 | ✓ 可 |
| 🔴★🔵 | 2 | ✕ 不可 |
| 🔴🔴★ | 2 | ✕ 不可 |

### 消去後の補充

- WCが消去されたマスには **通常ブロック** が補充される（WCとして補充されない）
- 次のWCはタイマーのしきい値で再び出現する

### 外観

- レインボーグラデーション（`hue-rotate` アニメーションで色相が回転）
- 中央に ★ マーク
- 疑似要素（ハイライト・シャドウ）は無効化しグラデーションを明確に表示
- 出現時にポップ（スケール拡大）アニメーション

## 9. 手動シャッフル

### 概要

プレイヤーが任意のタイミングでグリッドをシャッフルできる機能。残り時間を消費するコストと引き換えに使用できる。

### 仕様

| 項目 | 内容 |
|---|---|
| ボタン配置 | chain-bar 右端「🔀 -10s」 |
| タイムペナルティ | 残り時間 -10秒 |
| 使用可能条件 | ゲーム中 かつ 残り時間が10秒超 |
| 使用不可条件 | 残り10秒以下 / カウントダウン中 / アニメーション中 |
| チェーン選択中 | ボタンを非表示（誤操作防止） |
| 使用時の挙動 | チェーン・コンボをリセット → 時間減算 → 自動シャッフルと同じ演出を実行 |

---

## 10. 詰まり検出・シャッフル（自動）

### 判定ロジック

消去後、以下の条件を満たすチェーンが1つも作れない場合を「詰まり」と判定する。

- 通常ブロックを起点として BFS（幅優先探索）
- WCは任意の色として通過可能（WC通過後は異色ブロックへも進める）
- 通常ブロックへの到達数が3個以上 → 有効手あり

### シャッフル処理

- Fisher-Yates アルゴリズムでグリッド全体をシャッフル
- シャッフル後も詰まっている場合は最大3回再試行
- 3回試行後も解消しない場合は全マス完全ランダム再生成
- シャッフル発動時に「🔀 SHUFFLE!」バナーをアニメーション表示
- **タイムアップ後（timeLeft = 0）はシャッフルを実行しない**

## 11. コンボシステム

- 直前の消去から **3秒以内** に次の消去をするとコンボカウントが加算
- 3秒以内に消去がない場合はコンボがリセット
- コンボ倍率: 2コンボ目から +30%ずつ、最大×3.0倍

| コンボ数 | スコア倍率 |
|---|---|
| 1 | ×1.0 |
| 2 | ×1.3 |
| 3 | ×1.6 |
| 4 | ×1.9 |
| 5以上 | 最大×3.0 |

- HUDにコンボカウンター表示（加算時にパルスアニメーション）
- コンボ発生時に画面上部へ「n COMBO!」ポップアップ表示（コンボ数に応じて色変化）
- リザルト画面にゲーム中の最高コンボ数（BEST COMBO）を表示

## 13. ランキング

### 記録条件・内容

- スコアが0より大きい場合のみ登録
- 難易度ごとに **上位10件** をLocalStorageに保存
- 記録内容: 日時・プレイヤー名・スコア・ベストコンボ

### ソート順

スコア降順（同点の場合はベストコンボ降順）

### 表示

- タイトル画面の「🏆 RANKING」ボタンから表示
- ゲームオーバー画面の「🏆 RANKING」ボタンから表示
- 難易度タブ（EASY / NORMAL / HARD）で切り替え
- 1〜3位は金・銀・銅色でハイライト表示
- ランクイン時はゲームオーバー画面に「🏆 RANK n IN !」バッジを表示

### ランキング画面の閉じる動作

| 遷移元 | 閉じると |
|---|---|
| タイトル画面 | タイトル画面に戻る |
| ゲームオーバー画面 | ゲームオーバー画面に戻る |

## 14. データ保存

LocalStorageに以下のキーで保存する。

| キー | 内容 |
|---|---|
| `m3cc_best_{diff}` | 難易度ごとのベストスコア（数値） |
| `m3cc_ranking_{diff}` | 難易度ごとのランキング（JSON配列） |
| `m3cc_player` | プレイヤー名（文字列） |
| `m3cc_initialized` | 初回起動フラグ（`'1'`） |
| `m3cc_hint` | ヒント機能ON/OFFフラグ（`'1'`=ON / `'0'`=OFF、デフォルトOFF） |

## 15. UI・表示仕様

### HUD（ゲーム中常時表示）

| 項目 | 内容 |
|---|---|
| SCORE | 現在のスコア |
| COMBO | 現在のコンボ数 |
| TIME | 残り秒数 |
| BEST | 現在難易度のベストスコア |

- 残り時間が制限時間の33%以下: タイマー表示を警告色（オレンジ）に変化
- 残り10秒以下: タイマー表示を危険色（赤）に変化

### チェーンバー

- 選択中のブロック数をドットで表示
- 通常ブロック3個以上選択時: 「n個 — 先頭をタップして消去！」
- 3個未満: 「n個選択中… あとm個必要」

### ゲームオーバー画面

- SCORE（今回のスコア）
- BEST COMBO（今回のベストコンボ）
- BEST（難易度ベストスコア）
- NEW RECORD バッジ（ベスト更新時）
- RANK IN バッジ（ランクイン時: 「🏆 RANK n IN !」）
- ボタン: RETRY / TITLE / RANKING

### ビジュアルテーマ

- ネオン × ダークアーケード
- ノイズテクスチャ背景・スキャンライン効果
- ブロック発光（色ごとのグロー）
- フォント: Orbitron（見出し・数値）/ Noto Sans JP（日本語）

## 18. ヒント機能

### 概要

ゲーム中に一定時間操作がない場合、消去可能なチェーン候補を白グローでハイライト表示する補助機能。

### 仕様

| 項目 | 内容 |
|---|---|
| 初期値 | OFF |
| 設定場所 | 設定画面（⚙ SETTINGS）のトグルスイッチ |
| 保存先 | `m3cc_hint`（`'1'`=ON / `'0'`=OFF） |
| 発火条件 | ゲーム中（`running=true` かつ `_animating=false`）に5秒間操作がない |
| 表示内容 | ランダムに選んだ消去可能チェーン候補全セルに `.hint` クラス付与（白グローpulse） |
| 解除条件 | タップ/ドラッグ操作発生時に即時解除、タイマー再セット |
| 解除のみ（再セットなし） | `commitChain`/`cancelChain`/`endGame`/`goTitle`/シャッフル完了後はそれぞれ適切に制御 |
| 候補選定ロジック | `ChainManager` と同一の色判定ルール（WC含む）でBFS探索、通常ブロック3個以上到達できるグループをリストアップしランダム1件選択 |
| バックグラウンド対応 | タブ非表示時にタイマーをリセット、復帰時に再スタート |
| WC出現時 | グリッド変化により古い候補を破棄 |
| `prefers-reduced-motion` | pulseアニメーションを静的な白グローに自動フォールバック |
